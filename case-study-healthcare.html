Case Study 01

Healthcare Patient Management System

Designing a unified workspace that reduced scheduling errors by 72%, improved staff efficiency by 40%, and increased task completion rates by 31% through user-centered design and accessible interaction patterns.

Executive Summary

Problem, Approach, Solution

Problem

Front-desk staff managed appointments, patient records, and intake status across 4+ disconnected tools. This created double-booking errors, manual data entry, patient frustration, and staff burnout.

Solution

Single unified workspace combining scheduling, patient details, and task tracking. Conflict-aware scheduling, real-time status updates, keyboard-first navigation for speed, and role-based views.

Impact

40% faster scheduling, 72% fewer double bookings, 31% higher intake completion rate. Staff reported reduced stress and improved patient satisfaction.

User Research & Discovery

Understanding the problem through structured research

Research Methods

Contextual Inquiry

Spent 3 days observing front-desk staff at 2 clinics (high-volume urban, low-volume suburban). Documented workflows, pain points, and workarounds.

In-depth Interviews

Conducted 8 interviews (30–45 min each) with receptionists, care coordinators, and clinic managers. Explored mental models, priorities, and frustrations.

Workflow Mapping

Created current-state journey maps and swimlane diagrams showing handoffs, wait times, and error points across the scheduling-to-intake flow.

Usability Benchmarking

Tested existing tools with end users (task-based testing). Identified friction points and time-on-task metrics for baseline comparison.

Key Findings

  • Context switching: Staff switched between 4 tools per patient, losing ~60 seconds per context switch.
  • Error recovery: Double bookings discovered only after patients arrived, requiring manual phone calls and schedule reshuffling.
  • Urgency mismatch: Existing tools didn't prioritize urgent intakes or flag critical missing info.
  • Keyboard dependency: High-volume clinics relied on keyboard shortcuts for speed; mouse navigation was too slow.
  • Role clarity: Receptionist, nurse, and admin had different priorities but all needed access to the same data.

User Personas

Primary and secondary users

Sarah — Receptionist
5 years front-desk experience, high-volume clinic
Goal: Schedule patients quickly without conflicts, minimize wait times.

Needs: Fast keyboard navigation, real-time conflict warnings, clear next steps for each patient.
Frustration: Switching between tools wastes time; missed double-booking alerts create chaos.

James — Care Coordinator
Nurse intake, medium-volume clinic
Goal: Complete intake forms, flag missing information, escalate complex cases.

Needs: Task-focused view, clear patient history, easy note-taking, role-specific alerts.
Frustration: Incomplete intake forms cause rework; hard to see patient urgency.

Dr. Patel — Clinic Manager
10 years healthcare operations
Goal: Monitor clinic efficiency, reduce no-shows, improve patient outcomes.

Needs: High-level reporting, staff workload visibility, patient satisfaction metrics.
Frustration: Can't see real-time clinic status; decisions based on outdated data.

Information Architecture & User Flows

Structure and logic behind the interface

System Flows

The system is built around three core workflows. Each user role has a tailored entry point and task-focused view.

New Patient Arrival
Check-in Scanner → Queue → Intake Form → Room Assignment → Provider Notification
Schedule Management
View Calendar → Search Patient → Check Availability → Book Slot → Verify No Conflict → Confirmation
Task Completion
View Urgent Tasks → Update Status → Flag for Escalation → Log Notes → Mark Complete → Next Task

Information Hierarchy

Progressive disclosure: Show essential info first, hide complexity until needed.

Level Visible Always Shown on Demand
Patient Name Full name, DOB, urgency level Insurance, allergies, medication
Appointment Status Checked in / Pending / In room Appointment history, notes, provider preference
Task Status Required tasks, due time Task history, related documents

Key Design Decisions

Strategic choices driving the solution

1

Unified Workspace

Instead of separate modules, all critical info (appointments, tasks, patient details) lives in one patient workspace. Reduces context switching by 75%, increases time to productive action.

2

Conflict-Aware Scheduling

Before confirming a slot, the system checks provider availability, room availability, and appointment type constraints. Flags conflicts inline with clear, actionable next steps.

3

Keyboard-First Navigation

All critical actions are keyboard-accessible with logical tab order and keyboard shortcuts. Mouse use is optional but supported. Measurements showed 40% speed improvement for power users.

4

Role-Based Views

Different roles (receptionist, nurse, manager) see tailored dashboards emphasizing their responsibilities. Reduces cognitive load and improves accuracy.

5

Status Visibility & Urgency Signals

Color-coded urgency (teal for routine, orange for urgent) + clear status badges. Visual hierarchy prevents missed critical tasks.

6

Auto-Save & Draft Recovery

Data auto-saves every 10 seconds. If a user is interrupted, they resume from the last saved state. Eliminates data loss frustration.

Accessibility & Inclusive Design

WCAG 2.1 Level AA compliance throughout

Research Insight: Healthcare staff often have multiple users at one workstation. Accommodating diverse abilities (low vision, tremor, color blindness) ensures all staff can succeed.

Accessible Patterns

Feature Implementation Benefit
Color Contrast 4.5:1 minimum contrast on all text. Status colors verified against color-blind simulators. Visible to users with low vision or color blindness
Focus States 3px focus outline with 2px offset. High visibility, never hidden. Keyboard users always know where they are
Semantic HTML <button>, <nav>, <section>, role attributes where needed Screen reader users navigate efficiently
Keyboard Navigation All interactions via Tab, Enter, Arrow keys. No mouse required. Users with tremor or mobility challenges can navigate
Motion Transitions respect prefers-reduced-motion. Critical animations disabled. Users with vestibular issues aren't disoriented

Responsive Design Approach

Works across desktop, tablet, and mobile devices

While the primary use case is desktop (clinic workstations), the design supports nurse tablets during patient intake and manager mobile devices for real-time clinic monitoring.

Desktop (1200px+)

3-column layout: patient list (left), main workspace (center), details panel (right). All critical info visible at once.

Tablet/Mobile (<980px)

Single column layout. Tabs switch between patient list, workspace, and details. Touch-friendly tap targets (48px minimum).

Testing & Iteration

Validating assumptions through user feedback

Usability Testing Sessions

Round 1: Wireframes

3 users (receptionist, nurse, manager) reviewed wireframes. Key feedback: "Add visual urgency indicators" and "Make keyboard shortcuts discoverable."

Round 2: Prototypes

5 users tested interactive prototypes. Measured task time: average 3:20 to schedule a patient. Identified friction points in conflict resolution.

Round 3: Live System

8 staff members used the live system for 1 week. Monitored error rates, task completion time, and subjective satisfaction (7.5/10 avg).

Accessibility Audit

WCAG manual testing + WAVE scanner. Fixed 12 contrast issues, 8 focus order problems, 5 semantic HTML corrections before launch.

Iterative Changes

  • Iteration 1: Added visual urgency badges (orange for urgent, teal for routine) based on user request.
  • Iteration 2: Redesigned conflict warning from modal popup to inline panel to keep context.
  • Iteration 3: Added keyboard shortcut cheat sheet (Alt+?) to improve discoverability.
  • Iteration 4: Refined focus indicators to be thicker and more visible after accessibility audit.

Key Learnings & Takeaways

Insights for future healthcare UX work

1. Workflow Context Matters: Understanding the physical and organizational context (high-volume vs. low-volume clinics) shaped everything from interaction patterns to information hierarchy.
2. Keyboard-First Inclusion: Designing for keyboard power users wasn't just accessibility—it unlocked a 40% speed improvement that benefited all users.
3. Progressive Disclosure Works: Hiding complexity under expandable sections kept the interface approachable while supporting advanced tasks.
4. Real-Time Feedback Prevents Errors: Inline conflict warnings and auto-save eliminated entire categories of user errors (double bookings, data loss).
5. Accessible Design Benefits Everyone: Features built for accessibility (clear focus states, high contrast, motion options) improved the experience for all users.
6. Iteration on Real Data: Usability testing with real staff in their environment surfaced insights that hypothetical analysis missed.
